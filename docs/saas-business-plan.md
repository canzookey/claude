# SaaS 事業計画書

## 1. エグゼクティブサマリー

### 1.1 事業概要
本事業は、中小企業向けのクラウドベース業務効率化SaaSプラットフォームを提供する。プロジェクト管理、顧客管理（CRM）、請求・経理機能を一つの統合プラットフォームで提供し、中小企業のDX推進を支援する。

### 1.2 ミッション
「すべての中小企業にシンプルで強力なデジタルツールを届ける」

### 1.3 ビジョン
2030年までに国内中小企業向けSaaS市場においてトップ3のポジションを確立する。

### 1.4 主要KPI目標（3年計画）

| 指標 | 1年目 | 2年目 | 3年目 |
|------|-------|-------|-------|
| ARR（年間経常収益） | 1.2億円 | 4.8億円 | 12億円 |
| 有料顧客数 | 200社 | 700社 | 1,500社 |
| 月次解約率（Churn Rate） | < 3.0% | < 2.0% | < 1.5% |
| NRR（売上継続率） | 105% | 115% | 125% |

## 2. 市場分析

### 2.1 市場規模
- 国内SaaS市場規模: 約1.8兆円（2025年）、年率13%成長
- 中小企業向けセグメント: 約4,500億円
- ターゲット市場（SAM）: 約800億円（従業員10〜300名規模）

### 2.2 市場トレンド
1. **DX推進の加速**: 政府のデジタル化推進政策により中小企業のIT投資が増加
2. **クラウドシフト**: オンプレミスからクラウドへの移行が加速
3. **統合プラットフォーム需要**: 複数ツールの乱立による非効率を解消したいニーズ
4. **AI活用の普及**: 業務自動化・データ分析へのAI統合が差別化要因に

### 2.3 競合分析

| 競合 | 強み | 弱み | ポジション |
|------|------|------|-----------|
| A社（大手SaaS） | ブランド力、機能豊富 | 高価格、カスタマイズ性低い | エンタープライズ向け |
| B社（CRM特化） | CRM機能の深さ | 業務統合が限定的 | CRM特化 |
| C社（会計SaaS） | 会計機能の完成度 | プロジェクト管理機能なし | 会計特化 |
| **当社** | **統合性、価格競争力、AI活用** | **ブランド認知度** | **中小企業統合型** |

### 2.4 ターゲット顧客
- **主要ターゲット**: 従業員10〜100名の中小企業
- **業種**: IT/Web、コンサルティング、製造業、サービス業
- **課題**: 複数ツールの管理負荷、データの分断、高額なIT投資

## 3. プロダクト戦略

### 3.1 コア機能

#### フェーズ1（MVP: 0〜6ヶ月）
- プロジェクト管理（タスク、ガントチャート、カンバン）
- 基本CRM（顧客情報管理、商談管理）
- ダッシュボード・レポート

#### フェーズ2（成長期: 7〜12ヶ月）
- 請求書発行・経理連携
- ワークフロー自動化
- API連携（Slack、Google Workspace、Microsoft 365）

#### フェーズ3（拡大期: 13〜24ヶ月）
- AI分析・予測機能（売上予測、顧客離脱予測）
- 高度なレポーティング・BI機能
- モバイルアプリ（iOS/Android）

#### フェーズ4（成熟期: 25〜36ヶ月）
- マーケットプレイス（サードパーティプラグイン）
- 業種別テンプレート
- エンタープライズ向け機能（SSO、監査ログ、高度な権限管理）

### 3.2 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | React / Next.js / TypeScript |
| バックエンド | Node.js / NestJS |
| データベース | PostgreSQL / Redis |
| インフラ | AWS (ECS, RDS, S3, CloudFront) |
| CI/CD | GitHub Actions |
| モニタリング | Datadog / Sentry |
| AI/ML | Python / OpenAI API / 自社モデル |

### 3.3 差別化ポイント
1. **オールインワン統合**: 1つのプラットフォームで主要業務をカバー
2. **AI搭載**: 業務データからのインサイト自動生成
3. **価格競争力**: 個別ツール3つ分の半額以下
4. **日本市場最適化**: 日本の商習慣（請求書、見積書、稟議）に完全対応

## 5. マーケティング・営業戦略

### 5.1 Go-to-Market戦略

#### フェーズ1: PLG（Product-Led Growth）
- フリープランによるオーガニック獲得
- コンテンツマーケティング（SEO、ブログ、ホワイトペーパー）
- SNSマーケティング（X、LinkedIn、YouTube）

#### フェーズ2: セールス強化
- インサイドセールスチーム構築
- パートナーチャネル開拓（SIer、税理士事務所、商工会議所）
- ウェビナー・オンラインイベント

#### フェーズ3: エンタープライズ展開
- フィールドセールスチーム
- 大型案件対応（カスタム提案）

## 8. リスクと対策

### 8.1 主要リスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|---------|------|
| 大手SaaSの中小企業市場参入 | 高 | 中 | 日本市場特化、迅速な機能開発、価格競争力維持 |
| 顧客獲得コストの高騰 | 中 | 高 | PLG戦略強化、パートナーチャネル多様化 |
| エンジニア採用難 | 高 | 高 | 技術ブランディング、リモートワーク対応、SO制度 |
| セキュリティインシデント | 高 | 低 | SOC2取得、定期的なペネトレーションテスト、バグバウンティ |
| 景気後退によるIT投資縮小 | 中 | 中 | コスト削減効果の訴求、柔軟な料金プラン |

### 8.2 規制・コンプライアンス対応
- 個人情報保護法対応
- ISMAP（政府情報システムのセキュリティ評価制度）取得検討
- SOC2 Type II 認証取得（2年目）
- データセンター: 国内リージョン利用

## 10. 付録

### 10.1 用語集

| 用語 | 説明 |
|------|------|
| ARR | Annual Recurring Revenue（年間経常収益） |
| MRR | Monthly Recurring Revenue（月間経常収益） |
| ARPU | Average Revenue Per User（平均顧客単価） |
| CAC | Customer Acquisition Cost（顧客獲得コスト） |
| LTV | Lifetime Value（顧客生涯価値） |
| NRR | Net Revenue Retention（売上継続率） |
| Churn Rate | 解約率 |
| PLG | Product-Led Growth（プロダクト主導型成長） |
| CSM | Customer Success Manager（カスタマーサクセスマネージャー） |
| SO | Stock Option（ストックオプション） |

### 10.2 前提条件
- 月次顧客成長率: 15%（1年目）、12%（2年目）、8%（3年目）
- 平均契約ユーザー数: 10名/社
- 年間契約比率: 60%
- 粗利率: 75%〜90%（スケールに伴い改善）
- 為替レート: 考慮なし（国内市場前提）
